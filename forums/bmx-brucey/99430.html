Perhaps user rights/grants on the second server are not the same as those on the first.

According to the internet, the user "ODBC" is possibly a default username on Windows for MySQL, and maybe it is falling back to that because of permissions problems.

Thareh(Posted 2012) [#4]
Thanks for the replies! I solved the issue:
When using 'LoadDatabase()' with the username and password it works.
I was using the 'LoadDatabase()' without the username and password and then using the '.Open( Username, Password )' function.

The reason I was using the '.Open()' function was that I just couldn't get the connection to work - no errors just connection lost. And this was the only error message I could get :P Turns out the MySQL ping function doesn't work against that server, my guess is that it doesn't respond to pings due to security restrictions - In case anyone else has this problem! :)
