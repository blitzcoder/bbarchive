It's good to see that you update this product frequently. I didn't purchase yet, but I was thinking about if I should.

One thing that I miss (correct me if I am wrong) is a good UV Mapping tool in Tattoo. Currently I use a bunch of tools to do my mapping: modeling in 3dsmax, then add UV Coords in Lithunwrap 1.3, then paint the Texture in Photoshop.

I had serious problems with "hidden" triangles in Lithunwrap: when I use Box Wrap then some triangles become "invisible" because 2 Vertices share the same position, which results in Unmappable Triangles.

To fix this I wrote an App names UV-Doctor and I am thinking about to release this Tool for the BB Community, although the feature list is pretty short: The user can click Triangles on the (wireframe)displayed 3D Model to select a Triangle, then he can choose one of three Vertices and move this Vertexes UV Coords on the Texture. This way it's possible to fix the UVs of socalled "disobedient triangles".

However, I really would like to see Tattoo to offer some specialized Character UV Wrapping Capabilities. I think a lot of people have more problems with good UV (Un-)Wrapping than with painting the Texture.

As you say, the two are very much linked. Naff UVs will make it nigh impossible to paint in Tattoo (or anything else for that matter)!

Many people would like to see UV unwrapping. It's not too hard to do for basic stuff.

Problem comes when you need to save the UVs! Then you need to be able to save perfect versions of .B3D (not so much of a problem), .3DS, .X, .LWO, .OBJ files including their animation and hierarchy and support the various revisions. This causes MAJOR headaches and compatibility problems which is why I have kept out of it.

Only about 20% of Tattoo sales have come from Blitzers, so just doing the .B3D stuff is would mean excluding 80% of the users from a major function.

If there was an external UV only file format that all packages could read, then that would much easier.
