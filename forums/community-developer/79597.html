ok its been a while since i played with Lees converter
some strange behaviour going on with record pivot rotation

Problem : after a Record piv the mesh mashes up on conversion

it works on some models but not on others, this is what ive found

1: do your record pivot rotation

2: go into bone properties and zero out


Rest Rotation

Heading = 0
Pitch = 0
Bank = 0



this works for me it stops the mesh mangling up after record piv, i think lightwave leaves a load of crap in,
it brings the skelegons into layout at weird rotations at the start
... but zero out the Rest Rotation and it should be fine
