To be honest, I am hoping a few people might take to writing exporters while I am working on other features. The format is straightforward and well-organized, and it should be easy to at least write a program that scans 3DWS files.

Lighting is just about done, and there isn't really a good way to get the data out of the editor yet. I don't want to do lightmaps with .x files any more, because it's a lousy way of doing things. I'd rather use a .dbo exporter, or see some DBPro code to load maps, or a new DBPro plugin. For Blitz3D, a .b3d exporter would be good, or some .bb map loading code.

I'll get to it all eventually, but if anyone wants to do it themself, here's the info.
