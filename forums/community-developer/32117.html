jfk EO-11110(Posted 2004) [#9]
Yes, of course blitz can load animated 3DS, X or B3D. They don't use the MD2 Command Set, they use the standard Animation command set, such as Animate. You have to use LoadAnimMesh.

Personally I'd use any free Modeler (3dlinks.com), Lithunwrap 1.3 to set the UVs and CharacterFX to animate. This is a cheap solution too, but the Animation Part is more userfriendliy than Milkshape IMHO. Tho I got both, Milkshape and CharacterFX. Milkshape also has those good Game-Format Conversion Abilities.


jhocking(Posted 2004) [#10]
The file format conversion abilities of Ultimate Unwrap are much better (Ultimate Unwrap is the commercial "big brother" of Lithunwrap, available at www.unwrap3d.com ) Among other advantages, it can import .x and b3d file formats, and does so with skeletal animation intact.

My favorite free modeler is Wings3D (www.wings3d.com) although, because of some technicalities with how it understands the model data, you may want to tweak the model later (usually the model tweaks can be done in Ultimate Unwrap.)

Like jfk said, get CharacterFX for animating. Much better animation tools than Milkshape (eg. the Keyframe Editor) plus it supports blended vertex weights.


jfk EO-11110(Posted 2004) [#11]
I forgot to mention, Blitz can load animated .3DS and .X, but only Rotation, Scale and Move Animations. So the new DirectX8 Animations with their Skeletal deformation etc. are not supported. That's why you should use the .B3D Format. And yes, UltimateUnwrap is a must-have too (the sequel of lithunwrap). As far as I know it's the only tool capable of importing animated .B3D.
