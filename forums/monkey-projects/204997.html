EDIT #2:

Some notes about game development:
==========================================

Usually people ask what Frameworks are used, etc. I made the whole thing in Ted. Mojo, monkey, I'm not using any special modules except for my "fake" file system module for SaveState and a modified openURL function. I can't really think of anything else.

For a level editor I have a home-made one for this game made in Javascript and PHP.

Oh, for the courses I use Read/Write pixels so that I'm not drawing a couple hundred tiles each time, I generate the course from tiles, save the pixels and re-draw the new image each time. It was right after the release of these functions that I started porting this game - was just waiting, and they work amazingly well.

The only thing I hope for is the ability to turn on and off mojo image filtering in the future because on some Targets, Android, I get blurry artifacts around the edges of the tiles. I would like to turn OFF mojo image filtering while creating the course image, and then turn it back on to render the game in-play (if that makes any sense).
