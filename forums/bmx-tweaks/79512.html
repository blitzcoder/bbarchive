Space Fractal(Posted 2008) [#34]
I was began so tired of the find that still dosent allways catch strings (which need to typeing 4-5 times before it catch correctly), I diddecated to see I could fix that by my self.

It seen BlitzMax sometimes have problems with strings if $ is not ised if it have been defined (find$ example). That caused the string is outputet wrong to the variable or set wrongly to the textfeild the otherway.

Anyway here is what I did to get it work a least on my Vista x64 system in maxide.bmx:

Type TFindRequester Extends TRequester
	Field	findterm:TGadget
	Field LastFind$
	
	Method ShowFind(term$="")
		If term SetGadgetText(findterm,LastFind$)
		Super.Show()
		ActivateGadget findterm
	End Method

	Method Poll()
		Local	find$,data		
		find$=TextFieldText(findterm)
		Select EventSource()
			Case window
				If EventID()=EVENT_WINDOWCLOSE
					Hide
				EndIf				
			Case ok
				If EventID() = EVENT_GADGETACTION
					LastFind$=find$
					Hide
					PollSystem
					host.activepanel.Invoke TOOLFINDNEXT,find$
				EndIf
			Case cancel
				If EventID() = EVENT_GADGETACTION Then Hide
		End Select
	End Method

	Function Create:TFindRequester(host:TCodePlay)
		Local	seek:TFindRequester
		seek=New TFindRequester
		seek.initrequester(host,"Find",280,66,STYLE_OK|STYLE_CANCEL|STYLE_DIVIDER,"Find")
		CreateLabel("Find:",6,12,82,24,seek.window)
		seek.findterm=CreateTextField(88,8,ClientWidth(seek.window)-(88+6),21,seek.window)
		Return seek
	End Function
End Type
