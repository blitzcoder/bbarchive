Never had that problem with neither Toast 7 nor Toast 8.

Open Terminal and "cd" into Toast.app/..., locate the executable binary of the program and launch it from the command line - this might tell you what is going wrong.

Parallels installs a bunch of drivers on your system, so use Parallels' uninstall utility to remove the software.

I also did not like using Parallels with the BootCamp partition. I think it works much nicer with a local hard disk file, but the downside is that you need a second Windows license when you want to have both a native Windows and a VM installation.
