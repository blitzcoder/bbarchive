ok recieved a couple emails and to clear any further confusion, I am starting a small F.A.Q which will be moved onto the user forum as soon as mysql is setup on my host.

--------------FAQ-----------------------
Q: How many players are supported

A:The engine comes with blitzplay pro 1.0 I have talked with surreal the creator of Blitzplay and come to an agreement so that I may package the library with the Servant engine so the users dont have to use blitzplay lite. Your not only getting all these great modes,and an engine but also the ver 1.0 of blitzplay pro. So as of now there arnt any limits.

Q: What are the server requirements for running the games server?

A:To give you a basic idea of the server req I listed below the system I was using for the server with no lag experiance what so ever:

CPU : Amd athlon 500 mhz
RAM: 512 mb (SDRAM) * I recommend a high # of ram for processing speed

Storage: 20 gb Hard drive
Conenction: T1 (cable works just the samefor 2000 nodes)


Q: Is it seperate zones?

A: As far as the zones go this is what will be seen in a few updates from now or perhaps a large update. The game currently has VERY rough experimental zoning which will be replaced with endless mesh terrain. I hate playing mmo's and just when you think the world is huge you hit an invisible wall and poof your in another zone or something.

Q: Will there be a free version or a demo?

A: Yes I am finishing up the code.

Q: will a free version be able to be used for commercial?

A: Of coarse not the free version is released under the GPL lisence so anything you use form the engine must not be used for commercial products in anyway,shape or form. With that said remember anything made with this free version of the engine must be released opensource.

Q: Will the opensource engine keep up to date with the registered?

A: Nope I want to focus on giving registered users somthing great and I cant do that if im worried about 2 diffrent versions, however I might occasionally released an update for the GPL version but still no documents for it or help.

Q: What does the registered user get?

A: Well to start the are allowed to use models,code,media for any commercial release, they also will recieve model,and code updates as well as some documentation(help) kept up to date with new code and patches.

Q: What about making maps how do u do that?

A: I wrote a small 3d map editor called ServED which allows you to load models and place them then save to the world map format.

Q: I dont have blitz plus and the server was wrote in it? anyway I can get around that?

A: well even the beginner programmer can figure out what part of the server code is blitzplus related and which part isn't just strip out the blitzplus and your all set, I would like to note however if enough intrest is shown in the registered community I will write a Blitz3d version of the server although it may require more resources then the BLitzplus version.

Thats all I have right now I will add more as they come.
