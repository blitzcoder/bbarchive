GfK(Posted 2006) [#2]
It will update one frame each time you call UpdateWorld.

If you're using multiple UpdateWorlds for collision purposes, you should add a '0' parameter to all but one of them. This forces collisions only and won't update the animation.

jfk EO-11110(Posted 2006) [#4]
BTW you may also use delta-t as the parameter to make sure the animation runs with the same overall speed on machines with diffrent power/framerate. Something like:
old_t=t
t=millisecs()
delta#=float(t-old_t)/16.67
updateworld(delta)

You may have to initialize old_t before you start the main loop, eg: old_t=millisecs()-17
