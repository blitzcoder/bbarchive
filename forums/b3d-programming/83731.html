Ross C(Posted 2009) [#18]
Most of the animation is simply such commands and creating an animation state varaible such as:

animation_state = 0 ; 0 = static
                    ; 1 = walk
                    ; 2 = run


When the user presses the up arrow, to walk forward, you simply check to see if the up arrow key has been pressed, if so, then check to see if the walk animation is playing, if its not, use the animate command to animate him to walk. That's the basic way of doing it. Always keep a variable holding the current state of the animation, and what sequence it's animating.
