Brucey(Posted 2008) [#2]
Decide (at the start) how you intend to make the app localizable. Because you really don't want to be having to back-fit something half-way through.

There are a few modules available which can help you. The ones I know of first-hand are BaH.Locale, and wx.wxLocale. Obviously the wx module only works with wxMax.

They both use the ISO country codes, and handle more or less the full set of 190+ languages.
BaH.Locale can also show the language names natively - eg. Deutsch for German, etc.


Search the forums for others.

Here are some infos for multi language support in Logic Gui if not yet found:

> It offers all 190 iso languages and if it's not enough you can add your own.

> You can also spread your application to native speakers and collect from them language files you can import later without the need to type everything again.

> Adding a language in Logic Gui can take place at any time, so you are not limited to define 'something' in advance to be prepared.

> The current form can show whatever language is selected and you can adjust your gadgets to fit all possible variations.

> You can compile your code per language or with all languages together which can be switched during runtime.
