Hi Filax,

I just tried your code example with BlitzMax/MaxGUI v1.38, and the event is generated as expected when focus is lost. The difference you may be noticing, is that pressing 'Enter' in the text-field no longer causes the text-field to lose focus. This change was made in MaxGUI 1.34 after discussing with the community. The old behaviour can be emulated easily, using a hidden button with a BUTTON_OK flag.

See this thread/post for more info. :-)
