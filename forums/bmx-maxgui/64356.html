dmaz, not sure if you solved this on your own already, and apologies for not answering sooner.

Is it really a problem to overwrite the current parent block? Surely the new "rem" will be linked to original parent block anyway?

You can then search from the current position down to find an appropriate end point (EndRem). But you should only update those items that you find are already pointing to the previous parent. If you get to the end of that parent block, and still haven't found an end point, then you know that you've got a duf block (one with a start but no end).
