Kryzon(Posted 2010) [#5]
Since you said Ikaruga, that means that your background needs to be real-time.
You can use MiniB3D like everyone said - it'd be quick, [somewhat] painless and you get Max2D support to overlay your 2D sprites over the 3D background with blending, alpha, scaling, rotation etc.

You need to use the .B3D file format for that engine, though. You can use several software for the conversion, such as Gile[s], PaceMaker, Blender3D (+ GandalDF plugin), etc.

(note that PaceMaker is a mesh animator, but that still doesn't forbid you to export a static "environment" mesh).
