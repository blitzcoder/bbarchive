I was thinking about the docs problem, the lack of good comments and the lack of good code examples.

The solution to me is simple, if we create an online version(copy) of the docs we will be able to put them into a database and use them dynamiclly.

Imagine being able to goto the page for the command cos() for example, on the page for this command the community will have access to Submit and Example links, where they will be able to submit their own code Examples, these then get Authed by an admin and bingo the docs are auto updated, also means the docs will get updated a hell of a lot faster.

this wont be the only advantage to doing the online docs like this, there are many other examples of things that will be possible by doing it this way.

I would be willing to provide the SQL/Server but it might take a while to port the current docs to it.
