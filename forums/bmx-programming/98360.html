Linux does not care about extensions in the same way Windows does, and won't use them for something important like this. As a rule this means most binary executables there have no extension, because it serves no useful purpose. At any rate an extension is part of the filename, so if in doubt...look at the file?

Midimaster(Posted 2012) [#4]
I know this from opening html-documents, which are included with INCBIN: It does NOT work!. Because the system cares about the handling of the file in this case, it has to be in the directory of the system. The only way to do things like this, is to copy it first anywhere into the directory:

IncBin "test.html"
....
FileCopy "incbin::test.html" , "C:\temp\test.html"
OpenUrl "C:\temp\test.html"


by the way I agree with BladeRunner. Don't do that for levels.
