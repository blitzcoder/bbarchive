GfK(Posted 2014) [#1]
Don't ask me why I'm using ReadData. I just am.

Problem that's tripped me up this morning, is that this works:
ReadData t.x, t.y

...whereas this does not:
ReadData(t.x, t.y)

I habitually use parentheses like this, and as far as I can tell there is no reason for this to fail like it does:
Compile Error: Expecting ')' but encountered ','


[edit] This also works:
ReadData(t.x)
ReadData(t.y)



TomToad(Posted 2014) [#2]
ReadData is a command, not a function. When you use parenthesis, a return value is expected. The second case works because parenthesis is also used to group expressions such as (t.x+t.y)*5. (t.x) is a valid expression, (t.x, t.y) is not, hence the error.
