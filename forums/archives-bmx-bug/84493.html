SetGraphicsDriver GLMax2DDriver()

Graphics 1024,768

Local im1:TImage=LoadImage("a.png")
Local im2:TImage

While Not KeyHit(KEY_ESCAPE)
	Cls
	DrawImage(im1,0,0)
	If im2=Null Then
		im2:TImage=LoadImage("b.png")
		im2.Frame(0)   'load image in vram
	EndIf
	Flip
Wend


This will display im2, scaled to the size of im1. In directX mode it displays im1 as intended.

Hi,

TImage.Frame() was only ever intended for internal Max2d use, and removing it fixes the 'bug'.

However, it does highlight something slightly flaky within the glmax2d module which, now fixed, also fixes your code.

But I don't in general recommend using TImage.Frame()...
