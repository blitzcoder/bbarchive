GuShh(Posted 2014) [#3]
304 is an HTTP response code, it means the content was requested by the client, however since the content hasn't been modified, the server answered "no need to send you the data again, it's the same data". Should the data be modified, it would be sent again. This is merely to keep the cache up to date but also to limit unnecessary transfers.

It's not an error. You are just looking at the HTTP server Monkey uses to serve the data for html5, and that's one of the many responses it's sending out per requested object.

Hope that clears it out :)
