rsksenthil(Posted 2017) [#8]
Not the same case as discussed above but as a Monkey X newbie I had faced the below scenario.
What I have seen with downloading Pakz examples from https://github.com/Pakz001/MonkeyXExamples as zip file, unzipping the files and trying to run .monkey files is the following problem.
Almost all the .monkey files in the Pakz example files contain either the character "-" or " " (space) in the filename. This causes compilation errors.
When I change the filename to a name without "-" or " " they compile and run.

Example: "Generator - Stone 2.monkey" doesn't run but when renamed to "GeneratorStone2.monkey" or "Generator_Stone_2.monkey" it works correctly.
