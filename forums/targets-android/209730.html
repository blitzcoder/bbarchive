I recently installed Windows 10 on my PC.
While setting up Monkey X on it, I ran into a little issue when compiling for Android.
install:
     [echo] Installing C:\Program Files (x86)\Monkey X Pro\bananas\devolonter\matchup\matchup.buildv83c\android_new\bin\MonkeyGame-debug.apk onto default emulator or device...
     [exec] 	pkg: /data/local/tmp/MonkeyGame-debug.apk
     [exec] Failure [INSTALL_FAILED_UPDATE_INCOMPATIBLE]
     [exec] 3195 KB/s (209853 bytes in 0.064s)

BUILD SUCCESSFUL

I ran the "Fully automatic target installer" and chose "Android".
Once it was done, I opened the Android SDK Manager and chose API 13 and API 19 along with some other things like the USB Driver.
All installed correctly and Monkey X compiles things just fine.
So I'm really confused why I'm getting that error message.
I've tried deleting the build folder several times.


Hezkore(Posted 2015) [#2]
I solved it!

Turns out "Monkey Game" had been installed on the "Guest" account.
And even though I deleted it on my account, it was still on the "Guest" account, and I guess Monkey X was unable to replace it.
