Several things released(Posted 2006-01-17)
I'm not a good worklog writer, so here's a round up of what I used to release over 2005

After SlimShady I released TextureWobbler, a freeware tool to create seamlessy animated textures for "liquid substances", eg. Water etc. This tool can be found in the toolbox section.

I also released DrGreen, a tool dedicated for video compositioning, greenscreening. It also allows to add some filters to a video. The tool is using MCISendString to read the Frames from the source video and therefor it's pretty slow and even not very reliable. I am waiting for some people to complete a project for quick and accurate Frame Access from within Blitz. www.melog.ch/drgreen

Then I was busy with a nobudget amateur video production for several months. If you want so see the movies webpage, goto www.melog.ch/genom . Tho, don't expect too much, we're all beginners and did this just for fun. BTW Some effects were added using tools created with Blitz3D.

Late 2005 I found some time to release an Update for the CSP FPS engine: Version 5.5 added language support and a few more new features.

Traditionally I went underground around new year and completely concentrated on a new level design. Parallely to the level design I also added new features to the Engine, so finally I released the latest CSP Engine Update V6.0 and a playable demo yesterday, january the 16. 2006: www.melog.ch/dropper

New VIS Compiler(Posted 2003-01-31)
Added a VIS Compiler for lightmapped Level-Geometry (ie.Maplet Exports) to Code Archives. It splits a Mesh into several Clusters and occludes them dynamicly based on precalculated VIS-Data. The VIS Compiler uses the Unique Color Identifier System (UCI). Works best with HUGE Levels on crappy Computers. Will be implemented in the CSP FPS Engine Suite.

EDIT: I have removed it again from the Archives because it seems Maplet 2.0 will be released relativley soon and it will feature a native VIS Occlusions System that will be much more effective than mine. So it would be a bad idea to use mine.

CSP FPS Engine Suite(Posted 2003-01-22)
CSP FPS Engine Suite

Added Online Help.
http://melog.ch/dropper/help.html
http://melog.ch/dropper/



Interior Dropper Tools Suite contains several Parts:

A Level Editor. Load a Decorator- or Maplet-Level Mesh and add Interior like Chairs, Tables, Oil-Drums, Doors, Keys, Medi-Pacs, Ammo, Weapons, 3D-Sounds... One Goodie is the Preview Mode for Textures, Sounds and Meshes in the Fileselector.

A Waypoint Sequence Recorder. Walk around in FPP and record your Way. This saved Data can be used for a NPC-Waypoint Walk in the Game.

A Game Engine. It loads the edited Level and the Waypoint Files. It is simple yet but already gives you the Possibility to play trough a full Level, from Start to Exit.

These Tools are Open Source and you might have to tune them for your special Purposes.

Current Revision is V.0.8, featuring a working FPS Engine. Including the playable Demo Level "Operation Chateau Pregny".

Game Connector(Posted 2002-06-01)
I'm working on a Perl CGI-Script that will allow Players to choose from a List of hosted Games or to host a new Game and add their IP to the List. IPs are not visible to the Player (unless he's using packetsniffers) and everything is automated so the User only has to click the desired Game by Description.

The Script is running on a Webserver 24/7. Whenever 2 PPL want to play that game at the same time, they will meet eachother without the need to know eachothers IPs.

It works with a standartized Protocol and can be used for diffrent types of games.

Unfortunately there haven't been enough of beta testers online at the same time to do some serious stress tests which are required to localize some lil bugs.

I think I'll have to make a competition where PPL can win cash in my next stress test.

CSP Image Modeller(Posted 2002-03-19)
The CSP ImageModeller is a 3D Modelling Tool not unlike Photomodeller, but very simple to use. The user will have to load a Side- and a Frontview of a Person. He will then have to define the Vertices and Triangles based on these Images. There's still one hour work or so for one Face, but the Result shows that it's worth the effort. I know, there are several automatic Converters, but they wouldn't be that precise.
Please don't expect too much: it's a simple Tool.

Released first working Version (0.1). You can download it now (or maybe later due to limited Traffic on tripod).

This Program is in a very early stage, nonetheless it's already useful. By now it can only save Meshes as VRML. But there are many free .WRL to .3DS Converters, ie. 'Crossroads'.

BlitzMotion(Posted 2002-01-26)
BlitzMotion will be a low Cost Motion Tracking System. There are Plans for 2 diffrent Trackers: 2D and 3D. The 2D Tracker works with one Camera and is used for Speech Synchronizing and alike. The 3D Tracker uses 2 Cameras and can be used to record Model Animation such as Kung Fu etc.

The Project is in a very early stage, everything is a kind of a Work around at the Moment. I've got Webcam Images in BB with approx. 0.5 s Lag.
