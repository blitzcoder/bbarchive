Yes, I've got a first approach(Posted 2008-03-29)
I'm integrating a very small svn client to BLIde for BlitzMax users that want to have an updated BlitzMax distro.
The strongest 'point' on this svn integration is that everything is performed by BLIde in a transparent way. User's don't need to check out or update or rebuild the modules if needed or document their modules after an update, or update the BLIde modules database after an update is performed.

So in general terms, BLIde users will provide a username and a password, and the BlitzMax updater will automatically download any updated version, rebuild modules if needed, rebuild documentation if needed, etc, all in a single step. So... a small svn client, but focused on proper Max updates.
