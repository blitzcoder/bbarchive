MadMax(Posted 2005) [#12]
OK I will elucidate a bit. :)

I own all the Blitz products and think they are very good. And are perfectly capable for making awesome games. Blitz3D is also very good for 3D.

Things I like about PureBasic,

-Structures, once you define a structure you can use it as is, you can turn it into an array, a linked list (this is similar to Blitz types) only more powerful. You can create different linked lists that derive from the same basic structure, and extend it to make even more linked lists and use them all, if you want to, whithout them intefering one another (those familiar with Blitzmax will understand). Almost OOP without being OOP.

-Procedures, I like to be able to use static variables.

-Being able to make DLL's
-Use of ASM.
-Using the same language to do all, as PB is great for writing small utilities.
-Small exes, you end up falling in love with this feature.

And much more, but I don't want to make this a very long post. Of course PB is a bit more difficult to use than Blitz. All in all Blitz3D, Blitz+ and PureBasic are very powerful tools for making 2D games, and we should consider ourselves lucky to have these wonderful compilers.

I like BlitzMax very much, and I'm quite sure that it will soon (once a few issues are sorted out) become a very powerful language (already is). It will surely become a strong competitor to PureBasic.

I'd try PureBasic, it has a very generous demo, a great and helpful comunity. And why not buy the full version, it's very cheap; and will be useful anyway even if you use Blitz as your main development lang.
