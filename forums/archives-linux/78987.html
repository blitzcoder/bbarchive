MrTAToad(Posted 2008) [#2]
The easiest is Ubuntu, although all versions suffer a degraded graphical performance even with VMTools installed (if you can get it installed without having to install the complete set of linux code), so its always best to keep a real Linux machine around.

However using virtulisation does let you test things quickly and easily.

The worst so far as been OpenBSD - its nice to look at, but updating is buggy and error-prone.

And its no point trying Solaris - whilst Linux can be installed in it, its fiddly and you've got even more fiddling to do to get X11 up and running - so avoid!

Overall, I've tried :

CentOS
Mandriva 2008
Fedora 9
Ubuntu (two forms of it)
OpenSUSE
Xandros (if I remember correctly - the demo has expired now)

I failed with :

DSL- wouldn't install to hard drive
OpenBSD - sysinstall hung during updating process
OpenSolaris - Too fiddly and X11 requires more fiddling
MEPIS - Failed with kernel panic - not syncing (apparently doesn't like SCSI drives).
