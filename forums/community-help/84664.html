(This is an XP machine, btw.) I wanted to re-sort the list alphabetically, so I right-clicked on the list to bring up the Open/Explore//Delete/Rename//Sort by name/Properties menu.

Instead of the menu appearing, all that happened was that the main Start menu flickered. The O/E/D/R/S/P menu itself never appeared. I tried a reboot, but the same problem persisted. I even removed the custom skin I was using, going back to the default XP appearance settings; nothing worked.

It's not a huge issue in and of itself, but it prevents me from bringing up ANY right-click menus in the programs list. This means that if I want a shortcut for something on my desktop, I have to go the long way. I also can't re-sort the actual list of programs. This problem only occurs in the Start menu; everywhere else, right-clicks work fine, with or without the skin. Oh, and I have the My Documents folder in my Start menu as a menu instead of a link, for easy browsing; the right-clicks don't work there, either.

Any ideas?? @_@


GfK(Posted 2009) [#2]
Start > Control Panel > TaskBar and Start Menu > Start Menu tab > Start Menu/Customise > Enable Context Menus


Adam Novagen(Posted 2009) [#3]
Well, I had a look where you suggested, but there was no option to "Enable Context Menus." However, in the same place, when I clicked "Enable Darg and Drop," things started working again! O_o

SO, thanks Gfk, for pointing me to the right spot, and THANK YOU Microsoft, for labeling everything so CLEARLY!!! @_@
