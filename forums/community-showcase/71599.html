Actually a beta demo...
This little project started because a friend of mine wanted to write a game. he's an Oracle and Java programmer but this would be his real first attempt at a game with java. I asked him if he wanted to do a Java exercise or write a game. He said game... I said, get BlitzMax. He went "baaah", and started trying to do something in Java which he is very good at. So he started with this tutorial for an Opengl Java lib and played with it for a week and showed me the results. ok, good start but I told him if you want to make a game get BlitzMax! To prove my point I wrote the basics of this demo which included the aliens moving,firing and death. the player shooting and death. the particle engine too. that was done with Max from scratch (no extra libs, I didn't use any of my premade stuff(although I did have experience he didn't) in about 4 hours (including the first set of gfx) He was impressed and Mark got another sale!

Now it's about a 2 months later :P and I added a lot to it... the final 20% always takes the longest! I'm looking for any comments but mostly play performance. It does run a little slow right now on my test box(old duron laptop with S3 gfx)... I am in the process tuning the gfx and expect to make good progress with that. the only extra libs I now use on this are FMOD and a tweening lib I wrote for my other stuff.

so here it is... only 2 levels for this demo
cursor keys - move base.
space - fire
F5-F8 - modifies updates per second. F7=default.

Download

BTW: I am aware of the problem with the bonus screens...that screen is not finalized yet.

Most of the gfx are hand copies of the original Space Invaders. Most of the sound effects were taped (found on the net) from the original.
