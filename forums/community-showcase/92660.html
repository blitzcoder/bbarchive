Hi folks. I've been working with a little TCP/IP code (in Blitz3D) which basically consists of a Worms Server (which can be auto-started by the first client if it is not running) which can have up to 10 (maybe 20) clients attached to it.

The Server maintains a copy of where all the client worms are, their score (basically how long they've been alive) & if they've expired or not.

No collision detection invoked yet, my aspiration is to release a 'mopped up' version of the code to the coding archives so blitzers can see an example of how to hold 20 players data within a 512 byte packet (Using 20 bytes per player).
