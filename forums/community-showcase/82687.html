jfk EO-11110(Posted 2009) [#6]
Note using the Scancodes for Y/N questions will result in countryspecific incompatibilities, eg. in a german keyboard you't have to type "Z" for "Y"
. When you use Getkey instead then this should not be a problem. You may also combine Getkey and Keyhit to verify Ascii Chars.

In the Casino Minigames the 2D Text looks a bit rough, abe you should think about to use a pseudo-2D engine, that completely replaces the 2D Commands using 3D sprites or Quads. This will not only allow to use alpha and masking for smooth outlines, but also to scale and rotate 2D content.
There are some routines in the archives. Also make sure to read about "Pixelperfect" stuff, to prevent blurry Pseudo-2D.

The concept game is very fun. Only the graphics, esp. the sea, are still a little but too dark IMHO. Try to work with additional blendmode and funky fx, eg. particle effects.

Sound is nice.

Poker didn't show anything but a blank gradient - how did you make the gradient so smooth? Normally I get a Moire artefact with gradients, esp. if it's only eg. blue.

Some situations can only be handled by dropping one of the 2 extra chips by hitting left-ctrl. It's a bit odd when the game gives you a new chance for a level, but you can't finish it anyway because you've already spent the throwable chips. So maybe add new chips at level restart.

I played up to level 4 or so, tried to end the game with Esc (due to time pressure :) ), but I had to hack "q" to access the "exit game" option.

Very nice game, great intuitive game concept, still a bit dark and low contrast, but already great fun to play.
