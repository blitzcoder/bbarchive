sswift(Posted 2005) [#12]
Shag:
That file save issue has been fixed already. The problem was that the Blitzplus documentation is woefully inadequate as far as the load and save file dialogs go, and because of that I didn't format the string for the file types correctly, and as a result, not all files of a particular type were showing, and if you did not enter the extension the file dialog would not automatically add it, and would instead add the jpeg extension!

I have emailed you the corrected version.


Basically here was the problem:

If you want multiple file types in a file string, you might do it like so:

"*.jpg,.jpeg,.png,.gif"

For some reason Blitzplus requires you to have a * on the first extension in that list, but not on any of the others. If you have a * on any of the others, they won't make those files display in the file window!

Well I wanted a dialog that listed multiple file types, and the Blitzplus documentation doesn't tell you how to do that but someone here told me that you can use the semicolon to seperate the filenames to do that.

So I did something like this:

"*.jpg,.jpeg;.png;.gif"

But that was wrong. Though that displayed:
*.Jpg, .Jpeg
.Png
.Gif

In the file type dropdown box in the file dialog, it would not actually display all the png files in the current directory for example, or automatically append the png to the end of the filename.

It turns out the right way to do the file extensions is like so:
"*.jpg,.jpeg;*.png;*.gif"

So you have to put a * on the front of the first extension after each semicolon or else it doesn't work right.
