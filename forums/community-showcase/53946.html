Fancy an XMAS BONUS? Then download this festive puzzle game from Grey Alien Games. Line up the seasonal shapes against winter snowscape backgrounds, but don't chill out too much or the candle will burn out before you've cleared the grid. Swap shapes to score points with the help of exploding puddings, shooting stars and crazy crackers, topped off with some of your favourite Yuletide tunes. Come and join the festive fun!

Features:

# Seasonal fun

This beta actually only has 50 levels, I'm making the others right now! Read the readme.txt for interesting keys like B (Boss Key) and for the credits.

It's time to anounce my partnership with Indiepath on this one, they have been absolutely brilliant with this project, professional, supportive and helpful all the way, thanks!

OK, I know its a shape matching game, but I think it's nicely done with some original stuff; give it a go and see, it's very addictive! The theory is that this sorta game will sell well in what's left of the run up to Christmas and also, as it's my first commercial release, I didn't want to spend 4 years developing a 3D masterpiece only to see it flop ;-)

[EDIT] Please read the instructions as some people don't seem to get that you have to make all the tiles gold to complete a level.

Feedback of course is appreciated, especially of the constructive criticism type as opposed to utter flaming. Known Issues: The game doesn't like you pressing Ctrl+Alt+Del on Windows 2000 or XP Pro, but it's fine on other OSes, weird.

[EDIT] How to get DEBUG STATS: Press D in-game which brings up the debug stats. You can see the draw time (current/max) and the FPS (current/lowest). Sometimes the FPS lowest value really drops when redrawing alevel or using the menu so pressing D twice (off/on) resets it. The game should run at the same refresh rate as your video card e.g. 60Hz or 75Hz or 85Hz etc. If any lower I'd like to know about it! Can you post system specs?

wrote it in blitz plus all the effects like alphablended fonts/logos, screen fades, dims, brightness, sine waves, particles etc are software rendered. I used a quintuple buffering dirty rects system to totally optimise the drawing on low spec PCs.

Octothorpe. Can you run the game and press D which brings up the debug stats. You can see the draw time (current/max) and the FPS (current/lowest). Sometimes the FPS lowest value really drops when redrawing alevel or using the menu so pressing d twice (off/on) resets it. I have noticed on one other PC a very low FPS of 12-13 even though the draw time is perfectly reasonable and the PC seems good. This seems to be a VWAIT issue. Try going into the .ini file and changing FLIP FALSE from 1 to 0, this can solve VWAIT problems on some cards.
