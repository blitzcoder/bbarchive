The decision to begin this project came after realising how great it would be to have an application that brought all of a teams resources together more seamlessly. The application is being designed to help a team either over a large network/internet, or an indevidual developer keep his/her resources organised, documented, and safe from prying eyes.

The system will work by having a central "admin" pseudo server that controls general workgroup tasks, news, updates, and coordination. The application it's self will be Peer to Peer via UDP broadcasts. Each member in a work group(development team) will have access privileges(assigned by administrator from pseudo server) if desired, as well as full password protection and encryption on all files the developer wish's to keep safe.

The system will help coordinate a team with an integrated News, Messaging, and Automated file upload system. When the client starts up, it will ping all systems in the active projects work group, find new files, and then update it's self accordingly. Any system which is pinged by another system in the work group will test for new files on it as well.

A full host of configuration options will be available to actively control how the Asset Manager connects to other client/servers, including the ability to give indevidual files Access Level Requirements which a developer must have in order to download the file.

This application would not only allow a new horizon in blitz commercial application development, but in open source projects as well. Simply set your project to full acc/sharing, and anyone can download/modify the open source code/resources, re upload them, then allow you the central app developer to integrate the new source/media into the project at his leasure.

Most importantly, there is NO online storage required. Meaning you'll save cash on web hosting :).
