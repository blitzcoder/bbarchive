Chroma(Posted 2006) [#49]
Filax, if you're using single surface for the trees then there's no reason to hide the trees as they get farther away. You'll still pay the tris rendering if you "hide" them. The only way to cut down on the polys that way is to destroy the tree. Even if you move the tree -50000 on the y axis it will still be rendered because in a single surface system everything is being rendered no matter where it is. The benefit of a single surface system is that even though you're rendering everything regardless of where it's positioned, you're only rendering 1 mesh of 1000 trees instead of 1000 separate trees.

I hope you're going to credit Rob for using his single surface stuff...it would be nice.


Chroma(Posted 2006) [#50]
Also, you can use 1 mesh for all the leaf sprites for all the same type of trees. So if you have 100 oak trees, all the leaf sprites will count as 1 mesh. Then create a new layer for the oak trunks and lump them all together. So when you're rendering 100 oak trees you're actually only rendering 2 meshes. Rob's sprites will automatically point towards the camers which will be just like SpeedTree.

I'm going to finish up my tree system similar to SpeedTree (SuperTree) and release for all to use. It uses Rob's single surface code just like yours. I won't be charging for it.
