jhocking(Posted 2003) [#3]
Blitz does entity based frustrum culling. That is, objects offscreen are not rendered. Note however this is done on a per object basis; if any part of an object is onscreen then the entire object is rendered. Thus if your level is a single giant object (for example) the entire level is rendered even if the camera is way off to one side facing out and can only see a single wall.

Note also that this has nothing to do with occlusion. Even if an object cannot be seen because it is behind a wall it will still be rendered if it is within the camera view. For complicated scenes you may need to write some sort of culling system. The built-in frustrum culling is just a basic first step in reducing complexity of the scene just to what needs to be drawn.
