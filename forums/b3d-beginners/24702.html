WolRon(Posted 2003) [#7]
Gosub's are in for compatiblity with older BASIC's I believe. Functions are the new way of doing things.

I hear that Gosubs are actually faster at being called (which makes sense), but Functions are far more useful.

You can pass parameters to a function which you can't do with a Gosub.

You can return a value from a function which you can't do with a Gosub.

You can use default parameters with a function which you can't do with a Gosub.

Of course all of these things could be implemented using a Gosub with extra coding but isn't the point of having higher level commands to make coding easier for the programmer?

But with all that said, if your 'function' doesn't require any passed parameters and doesn't need to return any values, then you might as well use a Gosub for speed.
