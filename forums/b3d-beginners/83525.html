Hrm - maybe some Windows update could be the reason?
I just finished removing the harddisk from Thinkpad, built the hd back in Travelmate, deleted Thinkpad-Windows partition using Linux , reinstalled old Travelmate-Windows - first boot - installed recent Intel graphics drivers - reboot - installed Blitz3D and Update 1100 - works !?

First Windows Updates are running now, I'll see if Blitz3D stops working again...

EDIT: Everything works fine until ServicePack2 Update - then Blitz3D stops working (tries to compile, debug window opens - slowdown/freeze). Any ideas?

EDIT2: Service pack 3 does not make any difference - B3D crashes at compiling time.

Edit 3: Works!
Solution if somebody runs into similar issues:

- Fully updated WindowsXP
- In the Blitz3D-exe properties (right click)->compatibility activate the option 'Disable extended text services' (don't know the exact words in en)
- disable debug mode in Blitz3D

Not beeing able to use the debug mode may be a drawback for me - but finally B3D works on this little machine :)
Debug mode crashes wether the compatibility setting is changed or not. Strange.
