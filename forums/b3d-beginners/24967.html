semar(Posted 2003) [#2]
Both commands don't need to draw any image to perform overlap or collision test.

That means, if you don't draw your image on the screen, you can still perform collision check, if you need.

I guess that Blitz check for overlap/collision using a 'virtual' memory, so it does not display nothing on the screen.

These command work like the RectsOverlap(), RectsCollide() counterparts; you don't need to draw any rectangle to perform the related overlap,collision test.
