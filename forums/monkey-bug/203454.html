BlackD(Posted 2012) [#6]
This is a known bug with Flash in Windows 7, if Flash audio is playing and a USB sound device currently playing the sound is disconnected - it crashes the flash container. I haven't done much investigation into it but I have read about it in a few places but haven't done much looking into it because it's a client error and not really a software bug per se. Certainly at the Flash level, it's nothing we can correct in programming.

It may only happen in certain browsers (I know 32-bit Firefox on 64-bit systems is highly susceptible to it, and it can crash the entire browser not just a single tab or flash instance), and it may only happen with certain hardware (ie, drivers not releasing the audio correctly), but it's not something you're going to be able to fix in code, Origaming.

Edit: Just did some quick Googling - it can happen even with non-USB audio devices if you've got some smart drivers which try to change where the audio is playing once you unplug. Try using another browser and see if the same thing happens.
