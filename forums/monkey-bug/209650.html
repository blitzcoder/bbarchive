DomDom(Posted 2015) [#5]
Yep Strange. In fact, you are not doing exactly the same thing. You use another name for your app (DiddyApp)
Class DiddyApp Extends App

What Bono tries to do is using the exact same name (App) playing only with the 'namespace'
Class App Extends app.App

When, I change Bono to use another name (for example bonoApp) it works !!

As It works on IOS and HTML5, there may be a little inconsistency on android but this is not really a big problem as there are a lot of workarounds.
